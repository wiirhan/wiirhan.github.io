使用 Hugo 创建一个博客。

Hugo 是这么介绍自己的：

Hugo is the world’s fastest framework for building websites. It is written in Go.

安装 Hugo#

准备环境#

  1. 安装 Hugo
  2. 安装 Git

创建站点#

确认 Hugo 版本是 0.128.0 或以上。

hugo version

创建一个 ananke 主题的站点：

hugo new site my-blog
cd my-blog
git init
git submodule add https://github.com/theNewDynamic/gohugo-theme-ananke.git themes/ananke
echo "theme = 'ananke'" >> hugo.toml
hugo server

这样，一个使用 ananke 主题的站点就创建好了。

创建内容#

hugo new content content/posts/my-first-post.md

执行命令后，会在 content/posts/ 目录下创建一个 my-first-post.md 文件。

+++
title = 'My First Post'
date = 2024-01-14T07:07:07+01:00
draft = true
+++

这里的 draft 是 true，所以这篇文章不会在站点中显示。当你的文章写好后，可以将 draft 设置为 false。

修改这个文件后，运行 hugo server -D 命令，就可以在浏览器中看到这篇文章了。

hugo server -D

配置博客信息#

在 hugo.toml 文件中，可以配置博客的标题、描述、作者等信息。

baseURL = 'https://example.org/'
languageCode = 'en-us'
title = 'My New Hugo Site'
theme = 'ananke'

这里把 baseURL 设置为你自己的站点地址。

发布#

运行 hugo 命令，生成静态文件。

hugo

其他#

添加 .gitignore 文件，忽略 public 目录。

echo "public" >> .gitignore

部署在 GitHub Pages#

准备环境#

  1. 注册一个 GitHub 账号
  2. 安装 Git

步骤#

  1. 创建一个 GitHub 仓库，仓库名称为 your-github-username.github.io
  2. 把本地代码推送到 GitHub 仓库
  3. 在 GitHub 仓库中，点击 Settings，找到 Pages，在这里，你可以看到 Source 选项
  4. 在 Source 中选择 GitHub Actions。
  5. 在本地项目中创建一个文件
.github/workflows/hugo.yaml
  1. 在文件中添加以下内容, 并把 branch 名称和 Hugo 版本改成对应的
# Sample workflow for building and deploying a Hugo site to GitHub Pages
name: Deploy Hugo site to Pages

on:
  # Runs on pushes targeting the default branch
  push:
    branches:
      - main

  # Allows you to run this workflow manually from the Actions tab
  workflow_dispatch:

# Sets permissions of the GITHUB_TOKEN to allow deployment to GitHub Pages
permissions:
  contents: read
  pages: write
  id-token: write

# Allow only one concurrent deployment, skipping runs queued between the run in-progress and latest queued.
# However, do NOT cancel in-progress runs as we want to allow these production deployments to complete.
concurrency:
  group: "pages"
  cancel-in-progress: false

# Default to bash
defaults:
  run:
    shell: bash

jobs:
  # Build job
  build:
    runs-on: ubuntu-latest
    env:
      HUGO_VERSION: 0.137.1
    steps:
      - name: Install Hugo CLI
        run: |
          wget -O ${{ runner.temp }}/hugo.deb https://github.com/gohugoio/hugo/releases/download/v${HUGO_VERSION}/hugo_extended_${HUGO_VERSION}_linux-amd64.deb \
          && sudo dpkg -i ${{ runner.temp }}/hugo.deb          
      - name: Install Dart Sass
        run: sudo snap install dart-sass
      - name: Checkout
        uses: actions/checkout@v4
        with:
          submodules: recursive
          fetch-depth: 0
      - name: Setup Pages
        id: pages
        uses: actions/configure-pages@v5
      - name: Install Node.js dependencies
        run: "[[ -f package-lock.json || -f npm-shrinkwrap.json ]] && npm ci || true"
      - name: Build with Hugo
        env:
          HUGO_CACHEDIR: ${{ runner.temp }}/hugo_cache
          HUGO_ENVIRONMENT: production
          TZ: America/Los_Angeles
        run: |
          hugo \
            --gc \
            --minify \
            --baseURL "${{ steps.pages.outputs.base_url }}/"          
      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: ./public

  # Deployment job
  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    needs: build
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4

  1. 把本地代码推送到 GitHub 仓库

  2. 访问 https://your-github-username.github.io，查看博客

安装主题#

这里我选用的是 PaperMod 主题。

git submodule add --depth=1 https://github.com/adityatelange/hugo-PaperMod.git themes/PaperMod
git submodule update --init --recursive

需要更新主题时，运行以下命令：

git submodule update --remote --merge

最后，配置 hugo.toml 文件，设置主题。

theme = 'PaperMod'
